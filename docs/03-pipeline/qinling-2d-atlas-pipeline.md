# 秦岭切片 2D Atlas Pipeline

## 目标

当前原型不再把所有信息直接堆到 3D 场景里。新的主线是先建立一张严格地理坐标的二维地貌总览图，把 POI、水系、道路、地貌造型规则和叙事文案先在平面图上定准，再把它们投到 3D 游戏体验中。

## 架构原则

- 缩略图与 3D 游戏地形使用同一套严格地理坐标转换，不在地图投影上偷换比例。
- 非重点区域通过移动速度、镜头尺度、细节密度、事件密度压缩体验时间，不通过地图走形压缩距离。
- 2D Atlas 是信息源，3D 是体验层。3D 不再自己发明路线、水系、关隘和文案。
- 总览图默认只显示核心图层；军事、民生、人文等更细信息按需展开。

## 当前图层

- `landform`：关中平原、秦岭主脊、汉中盆地、米仓山/大巴山、剑门地形、成都平原。
- `water`：渭河、汉水、嘉陵江、褒水、斜水。
- `water` 的事实层分两级：默认地图保留人工校对过的主干骨架；全屏 Atlas 放大后加载 OSM/Overpass 命名水系证据层，逐级显示主要河流、支流和渠道。
- `road`：陈仓道、褒斜道、傥骆道、子午道、金牛道/剑门蜀道、米仓道、荔枝道意象。
- `city`：长安、咸阳、宝鸡/陈仓、汉中、广元、昭化、成都。
- `pass`：大散关、阳平关、剑门关。
- `military` / `livelihood` / `culture`：默认隐藏，作为后续专题层逐步补全。

## 3D 投放规则

- 水系先作为窄蓝色贴地带可见；3D 暂时只投放少量主干，避免性能和视觉噪声，全量 OSM 支流先留在 2D Atlas 中做校对和开发工作台。
- 古道在 3D 中只作为弱引导覆盖，不再用大面积黄色面抢夺地貌主体。
- 关隘和峡谷需要表现地形“收束”，不是只放一个点。
- 平原和盆地应更平滑、更开阔；山脊可以锐利，但不能全图等粒度尖峰。

## 下一步

1. 给 Atlas 增加图层开关 UI，让用户按地貌、水系、道路、城市、军事、民生、人文逐层查看。
2. 给每个 Atlas feature 增加文案卡片，点击后解释它的历史、人文、军事或民生意义。
3. 把水系进一步用于 3D 地形调色，让河谷和平原不只是一条线。
4. 为重点区域建立“造型粒度规则”：平原低频、山脊中高频、关隘/峡谷局部强化。
5. 在行动系统中让路线 affinity 影响速度、消耗和提示，而不仅是 HUD 文本。
