>
> **总数**：约 **260 POI**（含 8 政体 + 17 道 + ~70 城市 + ~35 山水 + ~80 古迹 + ~50 交通）。
>
> **Tier sizing**：
> - **T1 gravity** (250-300 行)：极大型，盛唐文明枢纽
> - **T2 large** (150-200 行)：节度治所 / 名山 / 重大古迹
> - **T3 medium** (100-130 行)：重要府州 / 二线名山 / 著名古迹
> - **T4 small** (30-60 行，几百字)：小县 / 小关 / 小景 / 远郊驿站

状态标记：
- ✅ done — doc 已落地
- 🔁 review — 待 codex review
- ⏳ pending — 排队中
- ❌ skip — 拍板放弃

# 写作执行策略

**批次 (batch) 推荐**：
1. **批次 A**：先写完所有 **T1 + T2 节度治所 / 都护治所**（~20 doc）—— 项目"骨架"先立
2. **批次 B**：T2 大山 + 主要古迹（~30 doc）—— 视觉锚点
3. **批次 C**：蜀道八条 + 关隘 + Tang 驿道（~30 doc）—— 交通骨架
4. **批次 D**：T3 medium 全部 (~100 doc) —— 充实
5. **批次 E**：T4 small 批量短文 (~60 doc) —— 收尾

**每批工作流**（per memory `feedback_workflow_claude_writes_codex_reviews.md`）：
1. 8 subagent 并行写
2. 抽检 codex review 1-2 个 / 每批
3. Claude 看 codex 报告，修订或重派
4. 通过则推 commit / 下批

**关闭条件**：批次 D 完后停下来跟 user review 整体覆盖度，决定 E 要不要做 / 要做哪些。
