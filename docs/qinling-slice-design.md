# 秦岭 - 关中 - 四川盆地 · 首版切片设计

## 区域目标

这块切片要完成一件事：

让玩家在 10 到 20 分钟内，身体上理解“为什么秦岭会把关中和巴蜀分成两个世界”。

## 当前层级定位

当前 `秦岭 - 关中 - 汉中 - 四川北缘` 不是最终局部细节地图，而是 `L1 全国游览层` 的局部试点。

这意味着：

- 它应该像 33 号远征队式的大地图一样，让玩家读懂大地貌、方向、路径入口和区域关系。
- 它不应该承担陈仓道、褒斜道、剑门关、汉中城等具体场景的全部细节。
- 缩略图、2D Atlas 和 3D 地形必须共用严格地理坐标；体验压缩通过移动速度、镜头尺度、细节密度和事件密度完成，而不是让地图坐标变形。

当前分辨率策略：

- L1 主地貌：按 `90m` 口径生成全国游览层。
- 重点修正：关中平原、汉中盆地、四川北缘、秦岭入蜀通道保留 `30m` 源数据修正权重。
- 低密度荒原：未来青藏高原、新疆大部分区域可按 `450m` 口径加工进入全国层。
- 当前运行资产：`193 x 241` 网格覆盖约 `420km x 560km`，约 `2.19km x 2.33km` 一个运行时采样点。

## 空间分层

建议把切片压缩成五个连续地带：

1. `关中北缘与黄土台塬`
2. `渭河平原 / 长安腹地`
3. `秦岭山口带`
4. `汉中盆地`
5. `入蜀谷道 / 成都平原`

## 体验节奏

### 第一段：开阔

- 玩家从关中平原出发
- 远处能看到横卧的秦岭
- 路线宽，移动轻快
- 首批收集强调“腹地”“养兵”“通达”

### 第二段：收紧

- 接近山前后，道路逐渐压缩
- 视野开始被山体切断
- 河谷和关口成为唯一合理路径

### 第三段：门轴

- 进入汉中后，突然没那么压迫
- 玩家会感到这里不是终点，而是转换枢纽

### 第四段：入蜀

- 重新经历山地与谷道挤压
- 最后抵达成都平原时，出现明显“松一口气”的空间反差

## 首批关键节点

- 长安 / 关中核心
- 渭河平原
- 秦岭主脊
- 散关或陈仓道意象节点
- 褒斜道意象节点
- 汉中盆地
- 剑门方向意象节点
- 成都平原

## 首批知识碎片主题

1. 为什么关中能成为核心腹地
2. 为什么平原能支撑大规模组织
3. 为什么秦岭是墙
4. 为什么过山不只是距离问题
5. 为什么山口比山顶更重要
6. 为什么汉中像锁眼
7. 为什么入蜀道路总被反复争夺
8. 为什么四川盆地易守难攻
9. 为什么盆地内适合长期经营
10. 为什么同样是十里路，平原与山道体验完全不同

## 首版禁忌

- 不要把知识点做成满地图问号
- 不要把 UI 做成任务清单游戏
- 不要让玩家在每个点都停下来读长文
- 不要一开始就做复杂 AI 战斗

## 首版成功标准

如果玩家在玩完后能自然说出下面三句话，首版就成立了：

- 关中真的像能养得起大城和大军的地方。
- 秦岭真的像一道把世界切开的墙。
- 入蜀之前的路真的让人理解“为什么这里难攻”。
